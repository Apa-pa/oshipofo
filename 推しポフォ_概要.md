# 推しポフォ プロジェクト概要

## これは何か

「推しポフォ」は、推しの構成比をドーナツチャートで可視化するフロントエンド中心の静的 Web アプリです。  
ジャンル別と作品別の 2 モードでポートフォリオを編集でき、保存・再編集・画像書き出し・共有フォーマット公開まで一通り行えます。

## 主な機能

- 推し比率をカテゴリ単位で編集
- 各カテゴリの内訳を子カテゴリとして管理
- `genre` と `works` の 2 モード切り替え
- ドーナツチャートと凡例のリアルタイム更新
- 比率順ソート
- 合計 100% への自動調整
- `localStorage` への保存と再読み込み
- PNG 画像として書き出し
- Firebase Firestore へのフォーマット公開
- 公開フォーマット一覧の参照と取り込み

## 画面構成

### 1. メイン編集画面 `index.html`

プロジェクトの中心画面です。

- 左側サイドバーでタイトル、名前、カテゴリ、比率を編集
- 右側でチャートと凡例を表示
- チャートをクリックすると子カテゴリのドリルダウン表示
- 下部アクションからクリア、保存、画像出力、フォーマット公開が可能

### 2. 保存済み一覧 `list.html` / `list.js`

ブラウザに保存したポートフォリオ一覧を表示します。

- 更新日時順でカード表示
- 既存データの再編集
- 不要データの削除
- 新規作成時は `oshipofo_load_id` をクリア

### 3. 公開フォーマット一覧 `formats.html` / `formats.js`

Firestore に保存された共有フォーマットを読み込む画面です。

- 公開済みフォーマットの一覧表示
- 任意のフォーマットを新規ポートフォリオとして取り込み
- 取り込み時は新しい保存データとして `localStorage` に追加

## 技術構成

- HTML / CSS / JavaScript の静的構成
- グラフ描画: `Chart.js`
- ラベル表示: `chartjs-plugin-datalabels`
- 画像化: `html2canvas`
- 永続化
  - ローカル保存: `localStorage`
  - 共有フォーマット保存: Firebase Firestore
- デプロイ前提
  - `CNAME` があるため、独自ドメインまたは GitHub Pages 系の公開運用を想定

## データの持ち方

### ローカル保存

主に次のキーを使っています。

- `oshipofo_saved_data`: 保存済みポートフォリオ配列
- `oshipofo_load_id`: 現在読み込む対象 ID

保存データには以下が含まれます。

- タイトル
- 名前
- 現在のモード
- `genre` / `works` ごとのカテゴリ構成
- 更新日時

### Firestore 公開データ

`formats` コレクションに以下を保存します。

- `title`
- `author`
- `createdAt`
- `categories`

公開時には、配布しやすいよう親カテゴリ・子カテゴリの比率を均等化して保存しています。

## コード上の特徴

- カテゴリは親子構造を持つ
- 親カテゴリに子がある場合、メインチャートでは親比率を子比率で按分して描画
- `autoAdjust()` で親カテゴリと子カテゴリの両方を 100% へ補正
- `refresh()` が UI 再描画の中心
- 状態は `modeData` と `categories` を軸に管理

## ファイル一覧

- `index.html`: メイン画面
- `style.css`: 全体スタイル
- `app.js`: メインロジック
- `list.html` / `list.js`: 保存済み一覧
- `formats.html` / `formats.js`: 共有フォーマット一覧
- `firebase-config.js`: Firebase 初期化
- `CNAME`: 公開用ドメイン設定

## 想定ユーザー価値

- 自分の「推しの内訳」を楽しく可視化できる
- SNS でシェアしやすい画像を作れる
- 他ユーザーのフォーマットを流用して気軽に作成を始められる
